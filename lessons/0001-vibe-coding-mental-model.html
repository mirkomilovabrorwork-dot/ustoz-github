Vibe-coding: coding emas, agentni boshqarish san'ati

Siz professional developer bo'lishingiz shart emas. Lekin professional natija uchun 4 ta rolni bilishingiz kerak: product owner, prompt writer, QA tester, va risk manager.

Time: 12 minutes Level: zero coding Win: first useful AI task prompt

Bugungi bitta fikr

AI bilan IT yechim qilishda sizning ishingiz kodni yodlash emas. Sizning ishingiz agentga nima kerakligini, qayerda ishlashini, qanday tekshirishini, va nimalarga tegmasligini aytish.

Oddiy formula: Goal + Context + Acceptance criteria + Verification + Risk boundary.

4 rol mental modeli

1. Product owner User uchun nimani yaxshilaymiz? Masalan: "Teacher upload qilgan video avtomatik summary olsin."
2. Prompt writer Agentga aniq task berasiz. "Fix it" emas, balki scope, criteria, va tekshiruv bilan.
3. QA tester AI qilgan ishni real user kabi sinaysiz. Button bosiladi, page ochiladi, error chiqmaydi.
4. Risk manager Payment, secrets, deploy, production data, auth kabi xavfli joylarni ajratasiz.

Yomon prompt vs yaxshi prompt

Yomon prompt:

Dashboardni yaxshilab ber.

Yaxshi prompt:

Goal: Dashboardda teacher video processing statusini tushunarli ko'rsin.
Context: apps/web dashboard page. User upload qilgandan keyin "processing", "transcribing", "ready", "failed" holatlarini ko'rishi kerak.
Allowed scope: dashboard UI va status display logic.
Do not touch: payment, auth, deploy, production secrets.
Acceptance criteria:
- Ready video "Ready" badge ko'rsatadi.
- Failed video retry yoki error text ko'rsatadi.
- Mobileda text overlap bo'lmaydi.
Verification:
- Run relevant web test or typecheck.
- Manually describe the user flow checked.

Mini practice

Quyidagi boxga o'zingiz xohlagan bir feature uchun prompt yozing. Masalan: "Telegram bot", "lesson dashboard", "online course landing page", yoki shu Ustoz app ichidagi kichik improvement.
