AI Vibe-Coding Cheat Sheet

Bir sahifalik eslatma: AI agentga yaxshi ish berish, natijani tekshirish, va xavfni boshqarish.

1. Agentga beriladigan yaxshi task

Context

Qaysi app, qaysi sahifa, qaysi user flow, qaysi fayl yoki bug?

Goal

User uchun nima o'zgaradi? "Button qo'sh" emas, "user payment statusni ko'rsin".

Acceptance criteria

Ish tugaganini qanday tekshiramiz? 2-5 ta aniq shart yozing.

Risk boundary

Nimalarga tegmasin? Payment, secrets, deploy, production data, auth.

2. Copy-paste prompt skeleton

Goal: [user nimani qila olishi kerak]
Context: [app/page/bug/flow haqida qisqa ma'lumot]
Allowed scope: [qaysi fayllar yoki qaysi feature]
Do not touch: [deploy, payment, secrets, unrelated refactor]
Acceptance criteria:
- [tekshiriladigan shart 1]
- [tekshiriladigan shart 2]
Verification:
- [qaysi test yoki manual flow yuritilsin]
Explain back in plain Uzbek/English what changed.

3. AI ishini tekshirish tartibi

  1. Read summary: Agent nima o'zgartirdi?
  2. Run narrow check: faqat tegishli test yoki typecheck.
  3. Try the user flow: browserda oddiy odam kabi bosib ko'rish.
  4. Check risk: auth, data, payment, secrets, deploy tegmaganmi?
  5. Commit only clean scope: bitta task, bitta tushunarli commit.

4. Red flags

  • Agent "tests passed" dedi, lekin test nomini yoki commandni aytmadi.
  • Oddiy UI fix uchun katta refactor qildi.
  • Production secret yoki deploy haqida o'zi qaror qildi.
  • Testni yashillashtirish uchun assertionni kuchsizlantirdi.
  • Foydalanuvchi flow bilan tekshirilmagan UI o'zgarish.
